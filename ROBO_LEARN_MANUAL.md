# 🤖 Robo Learn AI: คู่มือการใช้งานระบบสร้างโมเดล AI แบบ No-Code

คู่มือฉบับนี้จะช่วยให้คุณเข้าใจพื้นฐานการทำงานของแพลตฟอร์ม **Ai Teachstack** ตั้งแต่การเริ่มต้นลากบล็อกไปจนถึงการสั่งงานหุ่นยนต์จริงด้วยพลังของ AI

---

## 🚀 1. ขั้นตอนการทำงาน (Workflow)

กระบวนการสร้างระบบ AI สำหรับหุ่นยนต์แบ่งออกเป็น 4 ขั้นตอนหลัก ดังนี้:

1.  **การเตรียมข้อมูล (Input):** เลือกแหล่งที่มาของภาพ เช่น กล้องเว็บแคม, กล้องจากหุ่นยนต์ หรือชุดข้อมูลรูปภาพที่เตรียมไว้ (Dataset)
2.  **การเลือกสมอง AI (Model):** เลือกประเภทของโมเดลที่ต้องการใช้ (Detection / Classification / Segmentation) และตั้งค่าพารามิเตอร์ เช่น ความแม่นยำ หรืออุปกรณ์ที่ใช้ประมวลผล (CPU/GPU)
3.  **การประมวลผลและแสดงผล (Inference & Output):** เชื่อมสายเพื่อดูว่า AI มองเห็นอะไรผ่านหน้าจอ Monitor และดูตารางสรุปผลการตรวจจับ
4.  **การควบคุมหุ่นยนต์ (Robot Interaction):** สร้างตรรกะเงื่อนไข (Logic) เพื่อเปลี่ยนสิ่งที่ AI เห็นให้เป็นคำสั่งส่งกลับไปหาหุ่นยนต์ (เช่น ถ้าเจอคน > ให้หยุด)

## 📖 2. ความหมายของแต่ละบล็อก (Block Meanings)

### 📷 หมวด Image Data (Input)
*   **Webcam Stream:** ใช้กล้องจากคอมพิวเตอร์ของคุณเอง เหมาะสำหรับการทดสอบระบบเบื้องต้น
*   **Robot Camera:** รับภาพสดจากหุ่นยนต์ผ่านเครือข่าย (WiFi) โดยใช้รหัส Robot ID
*   **Test Images:** ใช้รูปภาพนิ่งที่มีอยู่ในโฟลเดอร์เพื่อทดสอบความแม่นยำของโมเดล
*   **Roboflow Dataset:** นำเข้าชุดข้อมูลที่ผ่านการ Label (ทำกรอบ) มาแล้วเพื่อเตรียมใช้ในการเทรน (Training)

### 🤖 หมวด AI Model (Brain)
*   **AI Object Detector:** บล็อกหลักสำหรับตรวจจับวัตถุและหาตำแหน่ง (Bounding Box)
    *   *Weights Source:* เลือกได้ว่าจะใช้โมเดลมาตรฐาน หรือโมเดลที่คุณเทรนเอง
    *   *Device:* เลือกระหว่าง CPU (ประหยัดทรัพยากร) หรือ GPU (ประมวลผลเร็ว)
*   **Image Classifier:** จำแนกประเภทของภาพทั้งภาพ (เช่น ภาพนี้คือแมว หรือ สุนัข)
*   **Instance Segmentor:** ตรวจจับวัตถุแบบละเอียดระดับพิกเซล (ระบายสีตามรูปร่างจริง)

### 📊 หมวด Results & Logic (Output & Robot)
*   **Live Monitor:** หน้าจอแสดงภาพสดพร้อมกรอบสี่เหลี่ยมที่ AI วาดทับ
*   **Detection Results:** ตารางแสดงรายการวัตถุที่เจอ พร้อมค่าคะแนนความมั่นใจ (Confidence)
*   **Logic & Condition:** ตัวตัดสินใจ (IF-THEN) กำหนดว่าถ้าเจอวัตถุชื่ออะไร ด้วยความมั่นใจเท่าไหร่ ให้ส่งคำสั่งอะไรออกมา
*   **Robot Data Sender:** บล็อกส่งคำสั่งกลับไปหาหุ่นยนต์ตัวจริงผ่านรหัส Robot ID

## 💡 3. ตัวอย่างการใช้งานแต่ละแบบ (Usage Examples)

### แบบที่ 1: ระบบตรวจจับวัตถุพื้นฐาน (AI Monitoring)
*เหมาะสำหรับ: การเรียนรู้เรื่องการทำงานของโมเดล AI เบื้องต้น*
*   **การเชื่อมต่อ:** `Webcam Stream` ➡️ `AI Object Detector` ➡️ `Live Monitor`
*   **ผลลัพธ์:** เมื่อส่องกล้องไปที่สิ่งของ AI จะวาดกรอบสี่เหลี่ยมและบอกชื่อวัตถุบนหน้าจอ

### แบบที่ 2: หุ่นยนต์เฝ้ายามอัจฉริยะ (Smart Guard Robot)
*เหมาะสำหรับ: การทำโปรเจกต์หุ่นยนต์ที่ตอบสนองต่อคน*
*   **การเชื่อมต่อ:** 
    1.  `Robot Camera` (ID: ROBOT_01) ➡️ `AI Object Detector`
    2.  `AI Object Detector` ➡️ `Logic & Condition` (Target: person, Command: STOP)
    3.  `Logic & Condition` ➡️ `Robot Data Sender` (ID: ROBOT_01)
    4.  `AI Object Detector` ➡️ `Live Monitor`
*   **ผลลัพธ์:** หุ่นยนต์จะวิ่งไปเรื่อยๆ แต่เมื่อกล้องของมันตรวจเจอ "คน" AI บนเว็บจะส่งคำสั่ง "STOP" กลับไปทำให้หุ่นยนต์หยุดเดินทันที

### แบบที่ 3: ระบบคัดแยกขยะ (Automatic Waste Sorting)
*เหมาะสำหรับ: การเรียนรู้เรื่องการจำแนกประเภท (Classification)*
*   **การเชื่อมต่อ:** `Webcam Stream` ➡️ `Image Classifier` ➡️ `Detection Results`
*   **ผลลัพธ์:** เมื่อนำขยะมาวางหน้ากล้อง AI จะแสดงผลในตารางว่าเป็น "Plastic", "Paper" หรือ "Metal" โดยไม่ต้องวาดกรอบตำแหน่ง

## 🛠 คำแนะนำเพิ่มเติม
- **Confidence Threshold:** หาก AI ตรวจจับผิดพลาดบ่อย ให้ลองปรับค่านี้ให้สูงขึ้น (เช่น 0.7 ขึ้นไป)
- **Robot ID:** ตรวจสอบให้แน่ใจว่า ID ที่กรอกในบล็อกและในโค้ด Python ของหุ่นยนต์ตรงกันเสมอ
- **Training:** หากต้องการตรวจจับวัตถุใหม่ๆ ที่ไม่มีในระบบ ให้ใช้บล็อกในหมวด Training เพื่อสร้างโมเดลของตัวเอง
